# Hyprland Lua Config Migration — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the Hyprland compositor config from deprecated hyperlang to Lua, validated incrementally, with `.conf` kept as a working fallback until the `.lua` tree is proven on real hardware.

**Architecture:** A templated `hyprland.lua.tmpl` entry point sets up `package.path`, then loads focused per-concern modules with `require()`: a shared `conf/helpers.lua` (the `o.*` ergonomic layer) and `conf/require_all.lua` (directory loader) first, then hyprsplit (`require("hyprsplit")`), environment/input/general/decoration/animations/autostart, the theme override module, all `conf/bindings/*.lua`, window rules, and `conf.d/*.lua`. Hyprland prefers `hyprland.lua` over `hyprland.conf` when both exist, so the old tree stays as a rollback path until cleanup.

**Tech Stack:** Hyprland 0.55.3 Lua API (`hl.*`, stubs at `/usr/share/hypr/stubs/hl.meta.lua`), hyprsplit Lua library (`require("hyprsplit")`), chezmoi Go templates, HyprDynamicMonitors (static-file profile), lua-language-server.

## Cross-Reference: Omarchy's completed migration (2026-06-24)

Omarchy (`~/Projects/_external/omarchy`, v4.0.0.alpha) has **already shipped** a full hyperlang→Lua migration. This plan was revised against their working tree, which resolves nearly every API question that was previously "verify on hardware". Patterns adopted from Omarchy:

- **`require()` + `package.path`** for all module loading (not `dofile()`), with a `require_all` directory helper (find+sort) — replaces the fragile `io.popen('ls ...')` glob.
- **An `o.*` helper layer** (`o.bind`, `o.window`, `o.exec_on_start`) for description-first, low-boilerplate config — see `~/Projects/_external/omarchy/default/hypr/helpers.lua`.
- **Theme = an `hl.config` override module** loaded after `general`, not a global `_theme` table threaded across scopes.
- **Confirmed dispatcher/rule syntax** baked directly into the translation reference below (no more guessing).
- **Plain shell `sleep N && cmd`** inside `exec_cmd` for delayed autostart — `hl.timer` is *not* required.

Authoritative Omarchy files referenced: `config/hypr/hyprland.lua`, `default/hypr/{helpers,require_all,paths,omarchy,windows,looknfeel,input,envs,autostart}.lua`, `default/hypr/bindings/{tiling-v2,media}.lua`, `default/hypr/apps/pip.lua`, `config/hypr/monitors.lua`, `default/hypr/toggles/voxtype.lua`.

> **Deviations from Omarchy (intentional):** we **keep hyprsplit** (independent 1-10 workspaces per monitor — Omarchy dropped it for global native workspaces); we keep the **voxtype recording submap** (multi-engine Parakeet/Moonshine/Whisper — Omarchy uses a single push-to-talk bind); and all entry points stay **chezmoi `.tmpl`** files.

## Module Loading Model (require + package.path)

The entry point sets one `package.path` covering three roots, then everything loads via `require()` (cached, real line numbers in errors, fresh state on every `hyprctl reload`):

```lua
local home   = os.getenv("HOME")
local config = os.getenv("XDG_CONFIG_HOME") or (home .. "/.config")
package.path = table.concat({
  config .. "/hypr/?.lua",       -- require("conf.environment"), require("conf.bindings.applications")
  config .. "/hypr/?/init.lua",  -- require("hyprsplit")  → ~/.config/hypr/hyprsplit/init.lua
  config .. "/?.lua",            -- require("dotfiles.extra-bindings"), require("themes.current.hyprland")
  package.path,
}, ";")
```

Module-name → file mapping (via the path above):

| `require(...)` | Resolves to |
|---|---|
| `require("conf.helpers")` | `~/.config/hypr/conf/helpers.lua` |
| `require("conf.require_all")` | `~/.config/hypr/conf/require_all.lua` |
| `require("conf.general")` | `~/.config/hypr/conf/general.lua` |
| `require("conf.bindings.applications")` | `~/.config/hypr/conf/bindings/applications.lua` |
| `require("hyprsplit")` | `~/.config/hypr/hyprsplit/init.lua` |
| `require("themes.current.hyprland")` | `~/.config/themes/current/hyprland.lua` (symlink) |
| `require("dotfiles.extra-bindings")` | `~/.config/dotfiles/extra-bindings.lua` |

**`conf.d/` has a literal dot in its name**, so module-name loading can't reach it. Follow Omarchy's toggles pattern: add `~/.config/hypr/conf.d/?.lua` to `package.path` and load with a `nil` prefix so bare filenames resolve.

All entries below are **confirmed against Omarchy's production Lua tree** (file cited) unless marked ⚠️.

### Core

| Hyperlang | Lua | Confirmed by |
|---|---|---|
| `env = VAR,value` | `hl.env("VAR", "value")` | `default/hypr/envs.lua` |
| `exec-once = cmd` | `o.exec_on_start("cmd")` (wraps `hl.on("hyprland.start", …)`) | `default/hypr/autostart.lua` |

| `monitor = name,mode,pos,scale` | `hl.monitor({ output = "name", mode = "mode", position = "pos", scale = N })` | `config/hypr/monitors.lua` |

| `windowrule = class X, float on` | `o.window("X", { float = true })` | `default/hypr/apps/*.lua` |

## Validation Method (used as the "test" in every task)

Hyprland Lua errors surface at load time. Per-task validation:

1. **Lua syntax:** `luac -p <file>` (or `luajit -bl <(chezmoi cat <target>) /dev/null`). Globals `hl`/`hs`/`o` are runtime-injected; a syntax check still catches the common errors.
2. **Template render (`.tmpl` only):** `chezmoi execute-template < source.tmpl` and `chezmoi cat <target>`.
3. **Live load:** with `hyprland.lua` present, `hyprctl reload`, then `hyprctl rollinglog | tail -40` and check for `[LUA]`/config errors. `hyprctl notify` shows error toasts.
4. **Functional:** exercise the specific binds/rules the task touched.

**Coexistence guarantee:** keep all `.conf` sources until Task 19. Hyprland loads `hyprland.lua` if present, else `hyprland.conf`. Rollback = `rm ~/.config/hypr/hyprland.lua && hyprctl reload`.

```lua
-- Directory loader + optional-require helper for the Hyprland Lua config.
local M = {}

local function shell_quote(p) return "'" .. p:gsub("'", "'\\''") .. "'" end

-- require() every *.lua in dir (sorted). module_prefix = nil when dir is on package.path.
function M.files(dir, module_prefix)
  local h = io.popen("find " .. shell_quote(dir)
    .. " -maxdepth 1 -type f -name '*.lua' -printf '%f\\n' 2>/dev/null | sort")
  if not h then return end
  for filename in h:lines() do
    local mod = filename:gsub("%.lua$", "")
    if module_prefix then mod = module_prefix .. "." .. mod end
    require(mod)
  end
  h:close()
end

-- require(module) only if its file exists (avoids hard error on optional files).
function M.if_exists(path, module)
  local f = io.open(path, "r")
  if f then f:close(); require(module) end
end

return M
```

- [ ] **Step 3: Write `conf/helpers.lua`** (defines global `o`)

```lua
-- Ergonomic helper layer for the Hyprland Lua config (global `o`).
o = o or {}

local function shell_quote(v) return "'" .. tostring(v):gsub("'", "'\\''") .. "'" end

-- o.bind(keys, description, dispatcher, options)
-- dispatcher: a hl.dsp.* value, OR a plain string (→ hl.dsp.exec_cmd).
function o.bind(keys, description, dispatcher, options)
  local opts = options or {}
  if description then opts.description = description end
  if type(dispatcher) == "string" then dispatcher = hl.dsp.exec_cmd(dispatcher) end
  hl.bind(keys, dispatcher, opts)
end

-- o.window(match, rules): string match → match.class; table match → merged into rules.match.
function o.window(match, rules)
  rules.match = rules.match or {}
  if type(match) == "string" then
    rules.match.class = match
  else
    for k, v in pairs(match) do rules.match[k] = v end
  end
  hl.window_rule(rules)
end

function o.exec_on_start(command)
  hl.on("hyprland.start", function() hl.exec_cmd(command) end)
end
```
