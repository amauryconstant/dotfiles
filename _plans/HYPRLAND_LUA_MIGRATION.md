# Hyprland Lua Config Migration — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the Hyprland compositor config from deprecated hyperlang to Lua, validated incrementally, with `.conf` kept as a working fallback until the `.lua` tree is proven on real hardware.

**Architecture:** A templated `hyprland.lua.tmpl` entry point sets up `package.path`, loads hyprsplit as a Lua library, sources the current theme's Lua color module, then `dofile()`s focused per-concern modules (environment, monitor, input, general, decoration, animations, window rules, 11 binding files) plus `conf.d/*.lua`. Hyprland prefers `hyprland.lua` over `hyprland.conf` when both exist, so the old tree stays as a rollback path until cleanup.

**Tech Stack:** Hyprland 0.55.3 Lua API (`hl.*`, stubs at `/usr/share/hypr/stubs/hl.meta.lua`), hyprsplit Lua library (`require("hyprsplit")`), chezmoi Go templates, HyprDynamicMonitors (static-file profile), lua-language-server.

## Verified API Translation Reference

All entries below are **confirmed** against the stub + official example + hyprsplit README. Items still needing a live test are marked ⚠️.

### Core

| Hyperlang | Lua |
|---|---|
| `env = VAR,value` | `hl.env("VAR", "value")` |
| `source = file.conf` | `dofile(os.getenv("HOME") .. "/.config/hypr/conf/file.lua")` |
| `exec-once = cmd` | `hl.exec_cmd("cmd")` inside `hl.on("hyprland.start", function() … end)` |
| `exec-once = sleep N && cmd` | `hl.timer(function() hl.exec_cmd("cmd") end, { timeout = N*1000, type = "oneshot" })` |
| `monitor = name,mode,pos,scale` | `hl.monitor({ output = "name", mode = "mode", position = "pos", scale = N })` |
| `monitor = …,transform,1` | add `transform = 1` to the table |
| `windowrule = match:class X, float on` | `hl.window_rule({ match = { class = "X" }, float = true })` |
| `windowrule = match:class X, size 800 600` | `hl.window_rule({ match = { class = "X" }, size = "800 600" })` ⚠️ verify string vs `{800,600}` |
| `windowrule = match:title T, opacity 0.9 0.8` | `hl.window_rule({ match = { title = "T" }, opacity = "0.9 0.8" })` ⚠️ |
| `layerrule = blur, namespace:^wofi$` | `hl.layer_rule({ match = { namespace = "^wofi$" }, blur = true })` |
| `submap = name … submap = reset` | `hl.define_submap("name", function() … end)` |

`hl.window_rule` takes an optional `name = "…"` identifier plus typed property fields (`float`, `size`, `center`, `pin`, `stay_focused`, `move`, `suppress_event`, `no_focus`, …). `match` is a table: `{ class, title, initialClass, initialTitle, xwayland, float, fullscreen, pin, workspace }`. Confirmed by the official example's `suppress-maximize-events` / `fix-xwayland-drags` / `move-hyprland-run` rules.

### Binds (confirmed from official example)

```lua
-- key string uses " + " separators
hl.bind("SUPER + Q", hl.dsp.window.close())
hl.bind("SUPER + Q", hl.dsp.window.close(), { description = "Close window" })   -- bindd

-- mouse bind: key string carries mouse:NNN, flag { mouse = true } REQUIRED
hl.bind("SUPER + mouse:272", hl.dsp.window.drag(),   { mouse = true })
hl.bind("SUPER + mouse:273", hl.dsp.window.resize(), { mouse = true })

-- scroll: mouse_down / mouse_up in key string, no mouse flag
hl.bind("SUPER + mouse_down", hl.dsp.focus({ workspace = "e+1" }))

-- flags: media keys
hl.bind("XF86AudioRaiseVolume", hl.dsp.exec_cmd("…"), { locked = true, repeating = true })

-- bindr (release) = bindd + release flag
hl.bind("SUPER + T", hl.dsp.exec_cmd("voxtype record stop"), { release = true, description = "Stop dictation" })

-- bind returns a handle (optional): local b = hl.bind(...) ; b:set_enabled(false)
```

`HL.BindOptions` fields: `repeating, locked, release, non_consuming, transparent, ignore_mods, dont_inhibit, long_press, submap_universal, click, drag, mouse, description, desc, device`.

## File Structure / Loading Order

`hyprland.lua.tmpl` orchestrates everything. Each module owns one concern and is independently `dofile`-able. The theme module returns a table; the entry point stashes it in a global `_theme` so `general.lua` can read it across `dofile` scopes (Lua globals persist across `dofile`).

```
hyprland.lua.tmpl
├─ package.path += ~/.config/hypr/?/init.lua      (so require("hyprsplit") resolves)
├─ _theme = dofile(themes/current/hyprland.lua)
├─ dofile conf.d/*.lua                             (voxtype-submap)
├─ {{ desktop }} dofile ~/.config/hypr/monitors.lua  (HyprDynamicMonitors output)
├─ dofile conf/monitor.lua                         (laptop fallback)
├─ dofile conf/environment.lua
├─ dofile conf/plugins.lua                         (hs.config)
├─ dofile conf/input.lua
├─ dofile conf/general.lua                         (reads _theme)
├─ dofile conf/decoration.lua
├─ dofile conf/animations.lua
├─ dofile conf/autostart.lua
├─ dofile conf/bindings/*.lua  (11 files)
├─ dofile conf/windowrules.lua
└─ pcall dofile ~/.config/dotfiles/extra-bindings.lua
```

## Validation Method (used as the "test" in every task)

Hyprland Lua errors surface at load time. Per-task validation:

1. **Lua syntax:** `luajit -bl <(chezmoi cat <target>) /dev/null` — or `luac -p`. (Globals like `hl`/`hs`/`_theme` are runtime-injected; a syntax check still catches the common errors.)
2. **Template render (`.tmpl` only):** `chezmoi execute-template < source.tmpl` and `chezmoi cat <target>`.
3. **Live load:** with `hyprland.lua` present, `hyprctl reload`, then `hyprctl rollinglog | tail -40` and check for `[LUA]`/config errors. `hyprctl notify` shows error toasts.
4. **Functional:** exercise the specific binds/rules the task touched.

**Coexistence guarantee:** keep all `.conf` sources until Task 18. Hyprland loads `hyprland.lua` if present, else `hyprland.conf`. Rollback = `rm ~/.config/hypr/hyprland.lua && hyprctl reload`.
